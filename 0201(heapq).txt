힙큐(heapq)
{
    개념 : 완전 이진 트리 기반의 최소 힙 자료구조(루트 노드의 값이 heapq 내부에서 가장 작은 값)

    사용 시기: 현재 존재하는 값 중 가장 작은 값을 원하는 경우에 사용

    sort와의 차이: sort의 경우는 모든 값을 정렬하지만 heapq의 경우에는 정렬이 아닌 루트 노드에 최소값을 두기 때문에

    정식 정렬과 같은 형태로 값을 보고 싶은 경우에는 heapq.heappop()을 이용하여  루트 노드를 제거해 준 후, 새로 들오온 루트 노드의 값을 보면 사용할 수 있다.

    결론 : 가장 작은 값을 순간순간 필요할 때 사용하면 좋으나, 정렬로는 사용하면 안된다.
}
